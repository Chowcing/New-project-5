后台管理工作台布局方向

请选择第一版更适合的结构。重点看信息密度、移动端可用性和管理员日常操作效率。

A. 分区总览 + 列表下钻
关键指标横向滚动：用户、交易、异常、审计
待处理用户
近期风险交易
按 Tab 查看用户 / 交易 / 审计列表

A. 稳妥增强

沿用现有四个 Tab，但把概览改成工作台，列表页增加筛选、详情和操作确认。实现风险最低。

B. 首页任务流 + 抽屉详情
今日待关注：禁用用户、异常交易、失败导入
快捷筛选 chips：未验证邮箱 / 大额支出 / OCR / 导入
主列表
详情抽屉

B. 推荐：任务型工作台

把后台从“查列表”升级成“发现问题、快速定位、执行操作”。适合管理工作台第一版。

C. 桌面化后台
侧边栏导航
工具栏和高级筛选
数据表格 + 批量操作

C. 重后台化

更像传统 Admin Console，桌面端强，但对当前移动优先产品和一次性范围更重。
